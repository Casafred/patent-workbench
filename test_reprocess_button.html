🔄 功能八 - 重新处理按钮测试指南

📋 功能概述

新增功能：重新处理按钮

允许用户使用当前图片状态（包括旋转后的图片）重新识别OCR和处理说明书，无需重新上传图片。

核心特性：
  • ✅ 使用当前图片状态（包括所有旋转操作）
  • ✅ 重新运行OCR识别
  • ✅ 重新匹配说明书内容
  • ✅ 清除旧缓存，强制重新处理
  • ✅ 支持AI模式和规则模式

🎨 按钮样式对比

颜色区分：

  • 绿色 - 开始处理（首次处理）
  • 橙色 - 重新处理（使用当前状态）
  • 红色 - 清空所有

🧪 测试步骤

步骤 1：首次处理
  1. 打开功能八页面
  2. 上传1-2张专利附图
  3. 输入说明书内容
  4. 点击 开始处理 按钮
  5. 等待处理完成，查看标注结果
步骤 2：旋转图片
  1. 在上传的图片列表中，点击旋转按钮（↶ 或 ↷）
  2. 旋转图片90度或180度
  3. 确认图片已旋转到正确方向
  4. 注意：此时缓存应该被清除
步骤 3：重新处理
  1. 点击 重新处理 按钮（橙色）
  2. 观察进度提示：正在重新识别OCR...
  3. 等待处理完成
  4. 验证标注结果使用的是旋转后的图片
步骤 4：修改说明书后重新处理
  1. 修改说明书内容（添加或删除标记）
  2. 等待1秒（防抖延迟）
  3. 点击 重新处理 按钮
  4. 验证新的匹配结果

✅ 预期结果

1. 按钮行为
  • ✅ 重新处理按钮显示为橙色渐变
  • ✅ 鼠标悬停显示提示：使用当前图片状态（包括旋转）重新识别OCR和处理说明书
  • ✅ 点击后按钮禁用，显示处理中状态
  • ✅ 处理完成后按钮重新启用
2. 缓存行为
  • ✅ 点击重新处理时，清除所有旧缓存
  • ✅ 控制台输出：🔄 重新处理：清除缓存，使用当前图片状态（包括旋转）
  • ✅ 处理完成后，保存新的缓存结果
3. 图片状态
  • ✅ 使用当前旋转后的图片进行OCR识别
  • ✅ 控制台输出每张图片的旋转角度
  • ✅ 标注结果反映旋转后的图片内容
4. 进度提示
  • ✅ 显示橙色进度指示器
  • ✅ AI模式：显示详细进度文本（正在重新识别OCR... → 正在分析附图... → ...）
  • ✅ 规则模式：显示简单进度文本（正在重新处理...）
  • ✅ 完成后显示：✓ 重新处理完成（橙色）

🔍 验证要点

关键验证点：
  1. 旋转后重新处理
    • 上传一张横向图片
    • 首次处理，记录识别到的标记
    • 旋转图片90度
    • 点击重新处理
    • 验证：标记位置应该随图片旋转而改变
  2. 缓存清除验证
    • 打开浏览器开发者工具 → Console
    • 点击重新处理
    • 查看控制台输出：应该显示缓存清除信息
    • 查看 localStorage：旧的 processing_cache_* 应该被清除
  3. 与开始处理的区别
    • 开始处理：绿色按钮，用于首次处理
    • 重新处理：橙色按钮，用于使用当前状态重新处理
    • 两者都会清除缓存并重新处理
    • 重新处理的进度提示颜色为橙色

🐛 常见问题排查

问题 1：点击重新处理按钮没有反应
  • 检查是否已上传图片
  • 检查是否已输入说明书
  • 查看浏览器控制台是否有错误
  • 确认按钮事件监听器已正确绑定
问题 2：重新处理后仍使用旧的图片
  • 检查控制台输出，确认使用的是当前图片状态
  • 验证 uploadedDrawings 数组中的 dataUrl 是否已更新
  • 检查旋转角度是否正确记录
问题 3：缓存没有被清除
  • 检查 clearCacheOnInputChange('重新处理') 是否被调用
  • 查看 localStorage，确认 drawing_cache_saved 标记被移除
  • 验证所有 processing_cache_* 键被清除
