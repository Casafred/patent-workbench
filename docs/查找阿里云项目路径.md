# 查找阿里云项目路径

你已经连接到服务器了，现在需要找到项目的实际位置。

## 在服务器上执行以下命令：

```bash
# 1. 查看当前目录
pwd

# 2. 列出当前目录的文件
ls -la

# 3. 查找项目目录（常见位置）
ls -la /www/wwwroot/
ls -la /home/
ls -la /opt/

# 4. 搜索 app.py 文件位置
find / -name "app.py" -type f 2>/dev/null | head -20

# 5. 搜索包含 patent 的目录
find / -type d -name "*patent*" 2>/dev/null

# 6. 查看 systemd 服务配置（如果有）
cat /etc/systemd/system/patent_system.service 2>/dev/null
systemctl status patent_system 2>/dev/null

# 7. 查看 nginx 配置（可能包含项目路径）
cat /etc/nginx/nginx.conf 2>/dev/null | grep root
cat /etc/nginx/sites-enabled/* 2>/dev/null | grep root
```

## 常见的项目路径：

1. **宝塔面板部署**：`/www/wwwroot/patent_system` 或 `/www/wwwroot/你的域名`
2. **手动部署**：`/home/patent_system` 或 `/opt/patent_system`
3. **root用户部署**：`/root/项目名`
4. **其他用户部署**：`/home/用户名/项目名`

## 找到路径后的回滚步骤：

```bash
# 假设找到项目在 /www/wwwroot/patent_system
cd /www/wwwroot/patent_system

# 查看是否是 git 仓库
git status

# 查看提交历史
git log --oneline -10

# 回滚到上一个版本
git reset --hard HEAD~1

# 重启服务（根据实际情况选择）
# 如果是 systemd 服务：
sudo systemctl restart patent_system

# 如果是宝塔面板：
# 在宝塔面板中重启 Python 项目

# 如果是手动运行：
# 先找到进程并杀掉，然后重新启动
ps aux | grep python
kill -9 [进程ID]
# 然后重新启动应用
```
