# 功能三：大批量处理

## 功能简介

大批量处理功能是专利智能工作台专为处理大规模专利数据设计的核心功能之一。该功能采用三步式工作流，包括生成请求文件、执行批处理和解析报告，能够高效处理海量专利数据，并生成结构化报告。

## 核心特性

### 1. 生成请求文件
- **Excel数据导入**：支持上传包含专利数据的Excel文件
- **多列配置**：可配置1-10列数据，适应复杂专利数据结构
- **API请求模板**：支持创建和保存自定义API请求模板
- **模型参数配置**：支持选择AI模型和调整温度参数
- **系统提示和用户提示**：可配置系统提示和用户提示，指导AI生成结果
- **输出格式定义**：支持自定义输出字段，确保结果结构化
- **JSONL文件生成**：自动生成JSONL格式的请求文件，适合批量处理

### 2. 执行批处理
- **请求文件上传**：支持上传生成的JSONL请求文件
- **Batch任务创建**：自动创建Batch任务，高效处理大规模数据
- **自动状态检查**：支持自动检查任务状态，无需手动刷新
- **断点续查**：支持恢复中断的任务，避免数据丢失
- **任务ID管理**：生成唯一任务ID，便于跟踪和管理
- **执行日志记录**：详细记录任务执行过程，便于调试和分析

### 3. 解析报告
- **原始数据匹配**：支持上传原始Excel文件，用于数据匹配
- **结果文件解析**：自动解析Batch任务结果文件
- **最终报告生成**：生成结构化的最终报告
- **报告导出**：支持导出为Excel格式，方便进一步分析
- **结果可视化**：直观展示解析结果，支持查看详细信息

## 操作步骤

### 1. 生成请求文件
- **步骤1**：上传专利数据Excel文件，选择工作表
- **步骤2**：配置分析列数量和对应列
- **步骤3**：配置API请求模板，包括模型选择、温度参数、系统提示和用户提示
- **步骤4**：定义输出字段和格式
- **步骤5**：生成并预览JSONL请求文件

### 2. 执行批处理
- **步骤1**：上传生成的JSONL请求文件
- **步骤2**：创建Batch任务
- **步骤3**：系统自动检查任务状态，或手动检查
- **步骤4**：等待任务完成，查看执行日志

### 3. 解析报告
- **步骤1**：上传原始Excel文件（用于数据匹配）
- **步骤2**：加载Batch任务结果文件
- **步骤3**：解析并生成最终报告
- **步骤4**：预览报告，确认无误后导出为Excel

## 注意事项

1. Excel文件大小建议不超过50MB，避免影响处理速度
2. 建议将相关数据放在同一工作表中，便于系统识别
3. 模板配置时，系统提示和用户提示应清晰明确，指导AI生成准确结果
4. 输出字段定义应符合JSON格式规范
5. Batch任务处理时间取决于数据量，建议耐心等待
6. 建议保存任务ID，便于后续恢复和查询
7. 结果文件较大时，解析可能需要一定时间
8. 最终报告生成后，建议及时导出和备份
