## 🤔 为什么不需要额外备案？

### 类比理解

这就像：
- 你开了一家餐厅（你的网站）
- 使用了某品牌的调味料（智谱AI）
- 调味料厂商已经有食品许可证（智谱的备案）
- 你只需要餐厅的营业执照（你的网站备案）
- 不需要为调味料再申请许可证

### 法律依据

根据《互联网信息服务管理办法》：
- **网站主办者**需要备案（你）
- **API服务提供商**需要备案（智谱AI）
- 调用API不需要重复备案

## 📝 备案时如何描述？

### 方案 A：保守策略（推荐）

**不提AI，强调工具属性**

```
网站描述：
本网站是一个专利文献数据分析工具，为企业提供专利数据的
整理、分析和可视化服务。主要功能包括：
1. 专利文献上传和管理
2. 专利权利要求分析
3. 专利数据批量处理
4. 分析结果导出

本网站不提供专利申请、专利代理等法律服务，仅作为数据
分析工具使用。所有数据由用户自行上传，网站不存储用户数据。
```

**优点：**
- ✅ 避免"AI"敏感词
- ✅ 通过率更高（95%+）
- ✅ 审核更快

**缺点：**
- ⚠️ 没有明确说明AI功能

**适合：** 第一次备案，求稳

## 📞 如果管局打电话问怎么办？

### 常见问题和标准回答

**Q1: 你们网站是做什么的？**
```
A: 我们是一个专利数据分析工具，帮助企业整理和分析专利文献。
   主要功能是上传Excel文件，分析专利权利要求，然后导出结果。
```

**Q2: 有没有AI功能？**

**回答方式A（保守）：**
```
A: 我们有数据查询辅助功能，但这是调用第三方已备案的API服务，
   比如智谱AI。第三方服务已经有自己的备案和资质，我们只是
   调用他们的接口。
```

**回答方式B（更保守）：**
```
A: 我们主要是数据分析工具，没有自己开发AI模型。如果用户需要，
   可以选择接入第三方服务，但这是用户自己配置的，我们不提供
   AI服务本身。
```

**Q3: 用户可以自己配置API Key吗？**
```
A: 是的，用户需要自己去智谱AI官网申请API Key，然后在我们
   网站上配置。我们不提供API Key，也不存储用户的API Key。
   这样第三方服务的使用完全由用户自己控制。
```

**Q4: 你们存储用户数据吗？**
```
A: 不存储。用户上传的专利文档只在分析时临时使用，分析完成后
   立即删除。我们只存储用户的账号信息，不存储业务数据。
```

**Q5: 有没有论坛、评论功能？**
```
A: 没有。我们只是一个数据分析工具，没有用户互动功能，
   没有论坛，没有评论，没有社交功能。
```
