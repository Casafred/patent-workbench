# 权利要求分析器 - 快速启动指南

## 🚀 5分钟快速上手

### 第一步：启动应用

```bash
# 方式一：使用部署脚本（推荐）
cd scripts
deploy_claims_analyzer.bat

# 方式二：直接运行
python run_app.py
```

### 第二步：打开分析器

在浏览器中访问以下任一地址：

**方式A - 从主页面进入（推荐）**
```
http://localhost:5000/frontend/index.html
```
然后点击顶部的 **"🔍 权利要求分析器（独立页面）"** 按钮

**方式B - 直接访问**
```
http://localhost:5000/frontend/claims_analyzer.html
```

**方式C - 测试页面**
```
http://localhost:5000/test_claims_analyzer.html
```

### 第三步：加载示例

1. 点击页面上的 **"📄 加载示例"** 按钮
2. 示例数据会自动填充到文本框中

### 第四步：开始分析

1. 点击 **"🔍 开始分析"** 按钮
2. 等待1秒，结果即刻显示

### 第五步：查看结果

页面会显示：
- ✅ 统计信息（总数、独权数、从权数）
- ✅ 权利要求列表（独权蓝色高亮，从权灰色）
- ✅ 可视化引证关系图

## 🎯 核心功能演示

### 功能1：识别独立权利要求

**输入：**
```
1. 一种智能手机，其特征在于，包括：
   处理器；
   存储器；
   显示屏。
```

**结果：**
- 显示为蓝色背景
- 标记为"独立权利要求"
- 在可视化图中显示为蓝色节点

### 功能2：识别从属权利要求

**输入：**
```
2. 根据权利要求1所述的智能手机，其特征在于，
   所述处理器为八核处理器。
```

**结果：**
- 显示为灰色背景
- 标记为"从属权利要求"
- 显示引用信息："引用: 权利要求 1"
- 在可视化图中显示为灰色节点，并连线到权利要求1

### 功能3：可视化样式切换

1. 在可视化区域找到"样式"下拉菜单
2. 选择不同样式：
   - **树状图**：层次清晰
   - **网络图**：可拖拽节点
   - **径向图**：环形布局

### 功能4：交互控制

- **🔍+**：放大视图
- **🔍-**：缩小视图
- **⚪**：重置缩放
- **🎯**：居中显示
- **📷**：截图保存

## 💡 使用技巧

### 技巧1：支持的编号格式

系统支持多种权利要求编号格式：

```
✅ 1.
✅ 1、
✅ 权利要求1
✅ 请求项1
✅ Claim 1
✅ 請求項1
```

### 技巧2：引用关系识别

系统能识别多种引用表达：

```
✅ 根据权利要求1
✅ 根据权利要求1或2
✅ 根据权利要求1至3
✅ according to claim 1
✅ according to claim 1 or 2
✅ 請求項1に記載の
```

### 技巧3：最佳可视化效果

- **少量权利要求（<10）**：使用树状图
- **中等数量（10-20）**：使用网络图
- **大量权利要求（>20）**：使用径向图

### 技巧4：快速截图

1. 调整好可视化样式和缩放
2. 点击📷按钮
3. 自动下载SVG格式图片
4. 可在其他软件中编辑

## ⚠️ 常见问题

### Q1: 为什么有些权利要求没有被识别？

**A:** 请确保：
- 每条权利要求有明确的编号
- 编号格式正确（如"1."或"权利要求1"）
- 文本完整，没有截断

### Q2: 如何处理复杂的引用关系？

**A:** 系统支持：
- 单个引用："根据权利要求1"
- 多个引用："根据权利要求1或2"
- 范围引用："根据权利要求1至3"

### Q3: 可以保存分析结果吗？

**A:** 目前支持：
- ✅ SVG格式截图
- ❌ Excel导出（计划中）
- ❌ 历史记录（计划中）

### Q4: 支持哪些语言？

**A:** 支持：
- ✅ 中文（简体/繁体）
- ✅ 英文
- ✅ 日文
- ✅ 韩文
- ✅ 其他主要语言
