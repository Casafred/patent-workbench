# models.json模型名称格式修复

## 问题描述
功能八AI说明书处理器报错：
```
Error code: 1211
Message: 模型不存在，请检查模型代码
```

## 根本原因
**模型名称格式不符合智谱AI官方API规范**

### 错误的模型名称（修复前）
```json
{
  "models": [
    "glm-4-flash",
    "glm-4.7-Flash",        // ❌ 错误：使用了点号和大写字母
    "glm-4-flash-250414",   // ❌ 错误：日期格式不是官方模型名
    "glm-4-flashX-250414",  // ❌ 错误：大写X和日期
    "glm-4-long"
  ]
}
```

### 正确的模型名称（修复后）
```json
{
  "models": [
    "glm-4-flash",          // ✅ 正确
    "glm-4-plus",           // ✅ 正确
    "glm-4-0520",           // ✅ 正确
    "glm-4-air",            // ✅ 正确
    "glm-4-airx",           // ✅ 正确
    "glm-4-long",           // ✅ 正确
    "glm-4-flashx"          // ✅ 正确
  ]
}
```

## 智谱AI官方模型名称规范
根据智谱AI官方API文档和用户提供的示例：

1. **全部小写**：所有字母必须小写
2. **使用连字符**：单词之间用 `-` 连接
3. **不使用点号**：不要使用 `.` 分隔版本号
4. **不使用大写**：不要使用大写字母（如Flash、X等）

### 官方模型列表
- `glm-4-flash` - 快速模型
- `glm-4-plus` - 增强模型
- `glm-4-0520` - 特定版本
- `glm-4-air` - 轻量模型
- `glm-4-airx` - 轻量增强模型
- `glm-4-long` - 长文本模型
- `glm-4-flashx` - 快速增强模型

## 为什么功能一（聊天）可以工作？
功能一使用的默认模型是 `glm-4-flash`（全小写），符合官方规范，所以可以正常工作。

## 修复内容
1. ✅ 移除 `glm-4.7-Flash`（不符合规范）
2. ✅ 移除 `glm-4-flash-250414`（不是官方模型名）
3. ✅ 移除 `glm-4-flashX-250414`（不是官方模型名）
4. ✅ 添加官方支持的模型：
   - `glm-4-plus`
   - `glm-4-0520`
   - `glm-4-air`
   - `glm-4-airx`
   - `glm-4-flashx`

## 修改文件
- ✅ `config/models.json`

## 测试验证
1. 刷新页面，清除浏览器缓存
2. 打开功能八
3. 选择任意模型（如 `glm-4-flash`）
4. 开启"AI说明书解析"开关
5. 点击"开始处理"
6. 验证API调用成功

## 技术要点
- 智谱AI API对模型名称大小写敏感
- 模型名称必须与官方文档完全一致
- 前端从 `config/models.json` 加载模型列表
- 后端直接使用前端传递的模型名称
- 不进行任何转换或映射
