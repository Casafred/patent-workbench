# 项目文件整理计划

## 🔍 现状分析

### 主要问题
- **根目录混乱**: 包含大量测试文件、部署脚本和冗余文件
- **文件结构不清晰**: 前端资源分散在多个位置
- **配置文件散落**: 配置文件分布在不同目录
- **文档管理分散**: 文档文件存在重复和分散的情况

### 目录结构现状
```
patent-workbench/
├── backend/                # 后端代码（模块化架构）
├── frontend/              # 前端资源
├── patent_claims_processor/  # 权利要求处理模块
├── js/                    # 冗余前端JS文件
├── docs/                  # 文档
├── tests/                 # 测试文件
├── output/                # 输出文件
├── test_data/             # 测试数据
├── .kiro/                 # 规格文档
├── .trae/                 # 开发工具文档
├── config/                # 配置文件
├── css/                   # 冗余CSS文件
└── 根目录测试文件*        # 大量测试文件
```

## 📋 整理计划

### 1. 目录结构优化

#### 创建新目录
- `scripts/` - 部署和管理脚本
- `tools/` - 开发工具和辅助脚本

#### 移动文件
- **根目录测试文件** → `tests/` 目录
- **部署脚本** → `scripts/` 目录
- **根目录JS文件** → `frontend/js/` 目录（如果需要）
- **根目录CSS文件** → `frontend/css/` 目录（如果需要）

### 2. 文件清理

#### 删除冗余文件
- 检查并删除重复的JS文件
- 清理过时的配置文件
- 删除不需要的备份文件

#### 整合配置文件
- 将配置文件统一到 `config/` 目录
- 清理过期的配置文件

### 3. 文档整理

#### 文档结构优化
- 统一文档到 `docs/` 目录
- 清理重复的文档文件
- 组织文档分类

#### 更新项目文档
- 更新 `README.md` 以反映新的目录结构
- 创建或更新 `docs/DIRECTORY_STRUCTURE.md`
- 整理部署相关文档

### 4. 最终验证

#### 功能验证
- 确保所有功能正常运行
- 验证测试通过
- 确认部署流程正常

#### 文档验证
- 确保文档与实际代码一致
- 更新所有相关文档路径
