## Phase 0: 基础设施准备

### 创建文件列表（全部为新文件，不影响现有功能）

**1. config/providers.json** - 服务商配置文件
- 定义智谱AI和阿里云百炼的配置
- 包含API地址、模型列表、功能支持等

**2. backend/services/llm/__init__.py** - 模块初始化

**3. backend/services/llm/base_provider.py** - 抽象基类
- 定义统一的接口规范
- 包含 complete、stream、get_model_list 等方法

**4. backend/services/llm/zhipu_provider.py** - 智谱AI实现
- 封装现有 ZhipuAI SDK 调用
- 实现基类定义的所有方法

**5. backend/services/llm/aliyun_provider.py** - 阿里云百炼实现
- 使用 OpenAI 兼容接口
- 支持思考模式、联网搜索等特性

**6. backend/services/llm/provider_factory.py** - 工厂类
- 根据配置创建 Provider 实例
- 统一管理 API Key 获取

**7. backend/services/llm/llm_service.py** - 统一服务层
- 提供统一的调用入口
- 模型名称映射
- 功能开关支持

### 设计原则
- 所有文件都是新增，不修改现有代码
- 保持与现有 `api_service.py` 的兼容性
- 通过功能开关控制是否启用新架构
